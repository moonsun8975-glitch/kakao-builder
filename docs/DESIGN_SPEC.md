## 2. 카드 구성 체계

### 2-1. 섹션 (시각 분류)
사이드바는 카드를 세 섹션으로 묶어 표시합니다. **"섹션"은 시각적 분류일 뿐**, 카드 자체에 고유한 표지/내지/아웃트로 속성은 없습니다. 사용자가 카드를 어느 섹션에든 자유롭게 배치할 수 있습니다.

| 섹션 | 영어 키 | 용도 |
|------|---------|------|
| 표지 | `cover` | 캐러셀의 첫 카드 (오프너) |
| 내지 | `inner` | 본문 카드 (혜택·상품·쿠폰 등) |
| 아웃트로 | `outro` | 마지막 카드 (카테고리·전체보기 등) |

### 2-2. 템플릿 ID

| ID | 이름 | 일반 용도 | 설명 | 커스텀 에디터 |
|:--:|------|----------|------|:--:|
| **A** | 브랜드형 | 표지 | 브랜드이미지 + 혜택 + 버튼 | ✅ |
| **B** | 가격강조형 | 표지 | 제품이미지 + 가격 + 버튼 | ✅ |
| **C** | 혜택형 | 내지 | 상단띠배너 + 배지 | ✅ |
| D | 제품리스트형 | 내지 | 상단띠배너 + 제품정보 | 범용 폼 |
| E | 쿠폰형 | 내지 | 쿠폰 최대 3개 | 범용 폼 |
| F | 쿠폰형 | 내지 | 쿠폰 최대 3개 (변형) | 범용 폼 |
| G | 이미지형 | 내지 | 브랜드이미지 + 할인율·원가·배지 | 범용 폼 |
| H | 프레임형 | 내지 | 브랜드이미지 + 배지 | 범용 폼 |
| H1 | 룩북형 | 내지 | 최대 2컷 조합 | 범용 폼 |
| H2 | 룩북형 | 내지 | 최대 3컷 조합 | 범용 폼 |
| H3 | 룩북형 | 내지 | 최대 4컷 조합 | 범용 폼 |
| I | 자유형 | 내지 | 제품이미지 + 제품명·원가·할인율 | 범용 폼 |
| J | 카테고리형 | 아웃트로 | 리스트 3·4·6·9개 | 범용 폼 |
| K | 리스트형 | 아웃트로 | 이미지 + 간략정보 | 범용 폼 |

> ✅ 커스텀 에디터가 완성된 A·B·C는 세부 설정이 가능하고, 나머지는 현재 범용 폼으로 동작. D~K 커스텀 에디터는 추후 개발 예정.

### 2-3. 카드 추가·이동 정책

- **최대 개수**: 10장 (초과 시 "+ 카드 추가" 버튼 회색 비활성화)
- **삽입 위치**: 선택된 카드 **바로 아래**. 선택이 없으면 배열 맨 뒤
- **섹션 귀속**: 새 카드의 섹션은 **삽입 위치 앞 카드의 섹션**을 따라감. 맨 앞 삽입이면 템플릿 기본 섹션 사용
- **번호 체계**: 사이드바·미리보기의 카드 번호는 **배열 순서대로 1, 2, 3...** 자동 부여. 삭제·이동·추가 시 즉시 재계산됨
- **CSV 불러오기**: 10장 초과분은 자동 제외되며 경고 표시

## 6. 이미지 업로드 플로우

### 6-1. 지원 포맷
- **업로드**: PNG, JPG, JPEG, AVIF, WEBP
- **AVIF 처리**: 브라우저 → Canvas → PNG 변환 후 업로드 (remove.bg가 AVIF 미지원)
- **URL 이미지**: 지원하되 **누끼 처리와 자동 컬러 추출 불가** (CORS 제한)

### 6-2. 이미지 타입 (표지 A 전용)

| 타입 | 설명 | 누끼 처리 | 배경 표현 |
|------|------|:--:|----------|
| 연출컷 | 배경 있는 이미지 | ❌ | 이미지 위에 그라디언트 오버레이 |
| 누끼컷 | 배경 제거된 이미지 | ✅ remove.bg API | 단색 배경 + 이미지 합성 |

### 6-3. 업로드 플로우 (케이스별)

#### 케이스 A: 표지 A 템플릿 / 누끼컷
```
1. 파일 선택 (또는 URL 입력)
2. 원본 이미지를 미리보기에 즉시 표시
3. _applyImgAndExtract() 호출
   → 이미지에서 주요 컬러 추출
   → 버튼 컬러(btnColor), 배경 컬러(bgColor), 그라디언트 컬러(gradColor) 자동 세팅
   → renderPreview(), renderEditor() 실행 (UI 갱신)
4. "누끼 처리 중..." 로딩 스피너 표시 (반투명 검정 오버레이 + 흰색 회전 원)
   └ 로더 위치: 에디터의 이미지 박스(imgBox_XX) 내부, z-index 10
5. remove.bg API 호출 (FormData로 이미지 전송)
6. 응답 도착
   ├ 성공: 누끼 처리된 PNG(투명 배경) 교체
   └ 실패: 원본 유지 + alert("누끼 처리에 실패했습니다. 원본 이미지를 사용합니다.")
7. 로딩 스피너 제거
8. renderPreview(), renderEditor() 재실행 (최종 이미지 표시)
```

#### 케이스 B: 표지 A 템플릿 / 연출컷
```
1. 파일 선택
2. 이미지 미리보기 표시
3. _applyImgAndExtract() 호출 (누끼 X, 컬러 추출만)
4. 그라디언트 오버레이 자동 생성 (이미지 중앙 + 평균 컬러 혼합)
```

#### 케이스 C: 표지 B 템플릿 (무조건 누끼)
```
1. 파일 선택 (B는 항상 "누끼 권장")
2. 로딩 스피너 표시
3. remove.bg API 호출
4. 응답 처리
   ├ 성공: 누끼 이미지 저장
   └ 실패: 원본 저장 + alert 경고 (가격강조형은 누끼 필수라는 안내)
5. 스피너 제거 → 미리보기 갱신
```

#### 케이스 D: 내지 C 템플릿 (연출컷 전용)
```
1. 파일 선택 → 즉시 미리보기 표시
2. 누끼 처리 없음 (배경에 연출컷 그대로 사용)
3. 드래그·핸들·휠로 위치/크기 조정 가능
```

### 6-4. 로딩 스피너 디자인
- **배경**: `rgba(0,0,0,0.45)` 반투명 검정 오버레이
- **크기**: 28×28px 원형 스피너
- **색상**: 흰색 반투명 테두리 + 완전 흰색 상단
- **애니메이션**: `nukkiSpin` 0.8초 linear 무한 회전
- **텍스트**: "누끼 처리 중..." (폰트 11px, 흰색)
- **위치**: 에디터의 해당 이미지 박스 내부에 absolute 오버레이
- **z-index**: 10

### 6-5. 이미지 위치 조정 도구

이미지 업로드 후 에디터에서 **"위치 조정"** 버튼 클릭 시 활성화됩니다.

| 조작 | 동작 |
|------|------|
| **드래그** | 이미지 이동 (마우스 이동량 / 0.5 배율 보정) |
| **8개 핸들** | 크기 조정 (코너 4개 + 엣지 4개) |
| **마우스 휠** | 확대/축소 (0.05 단위) |
| **완료 버튼** | 편집 종료 (필수 — 눌러야 편집 상태 해제) |

- **스케일 범위**: 0.1 ~ 6.0 (연출컷 최소 1.0, 누끼컷 최소 0.1)
- **경계 제약**: 연출컷은 `cover` 채움을 유지하는 범위로 이동 제한
- **편집 UI**: 파란 점선 테두리 + 8개 핸들 + 툴팁("드래그: 이동 | 핸들: 크기 | 휠: 줌")
- **편집 상태 저장**: `card.data._imgEditing` 플래그. 카드 전환·렌더링 후에도 복원됨

### 6-6. 초기화 동작
"초기화" 버튼 클릭 시:
- `_imgX = 0`, `_imgY = 0`, `_imgScale = 1` 로 데이터 리셋
- 미리보기의 transform도 동시에 복원
- 다른 카드로 전환 후 돌아와도 유지됨 (BUG-07 패치 완료)

## 7. 자동 컬러 추출 시스템 (표지 A 전용)

### 7-1. 추출 로직
이미지 업로드 시 두 가지 컬러를 분석합니다:

1. **평균 컬러** (`extractImgColor`): 전체 이미지를 16×16으로 축소 후 가장 많이 등장하는 컬러
   - 너무 밝거나(230+) 어두운(20-) 픽셀은 제외
2. **중앙 컬러** (`extractImgCenterColor`): 이미지 중앙 20% 영역의 평균 RGB

두 컬러를 혼합해서:
- **그라디언트 컬러**: 혼합 컬러 × 8% + 흰색 × 92% (연한 톤)
- **버튼 베이스**: 두 컬러 중 채도 높은 쪽 선택
- **버튼 컬러**: 버튼 베이스의 채도 +25%, 명도 -25% (HSL 변환)
- **배경 컬러**: 버튼 베이스 × 18% + 흰색 × 82%

### 7-2. 자동/수동 토글 (`_XXXAuto` 플래그)

각 컬러마다 "자동 모드" 플래그가 있습니다.

| 키 | 의미 |
|----|------|
| `_btnColorAuto` | 버튼 컬러 자동 모드 (기본 `true`) |
| `_gradColorAuto` | 그라디언트 컬러 자동 모드 |
| `_bgColorAuto` | 배경 컬러 자동 모드 |

**동작 규칙:**
- 사용자가 iro 피커로 컬러를 직접 변경하면 → 해당 플래그 `false` 로 변경 (= 수동 모드)
- 에디터의 "자동" 버튼을 클릭하면 → 플래그 `true` 로 복원 + `_autoXXX` 값으로 재세팅
- 이미지 타입 전환(연출컷 ↔ 누끼컷) 시: 플래그가 `true` 인 컬러만 재계산, `false` 는 사용자 설정 유지

### 7-3. UI 표시
- **자동 모드**: "자동" 버튼이 노란색 배경(`--y200`) + 카카오 브라운 텍스트
- **수동 모드**: "자동" 버튼이 회색 테두리 + 회색 텍스트
- **컬러 피커**: iro.js 원형 박스 + 슬라이더 + 스포이드 버튼 + HEX 입력

## 9. 외부 서비스

### 9-1. remove.bg (누끼 처리)
- **엔드포인트**: `https://api.remove.bg/v1.0/removebg`
- **인증**: X-Api-Key 헤더 (현재 코드에 하드코딩 — 🔴 보안 이슈)
- **파라미터**: `size: auto`
- **무료 한도**: 월 50장
- **실패 처리**: 원본 이미지 사용 + alert 경고

> ⚠️ **알려진 리스크**: API 키가 클라이언트 코드에 노출되어 있어 도메인 제한 또는 온디바이스 모델(`@imgly/background-removal`) 마이그레이션이 필요함.

### 9-2. iro.js (컬러피커)
- **CDN**: `https://cdn.jsdelivr.net/npm/@jaames/iro@5`
- **구성**: Box(HSV) + Slider(Hue)
- **크기**: 240px
- **스포이드 기능**: 업로드된 이미지에서 클릭으로 컬러 추출 (URL 이미지는 CORS로 불가)

### 9-3. Google Fonts (Pretendard)
- **링크**: `https://fonts.googleapis.com/css2?family=Pretendard:wght@400;500;600;700&display=swap`
- **굵기**: 400, 500, 600, 700
