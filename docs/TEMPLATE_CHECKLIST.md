# 새 템플릿 만들기 체크리스트

새 템플릿(D, E, F...)을 만들 때 이 체크리스트를 따라가세요.
`docs/COMPONENT_RULES.md`의 규칙을 준수하면 일관성이 유지됩니다.

## 📋 STEP 1: 기획

**디자인/기획 단계에서 결정할 것**

- [ ] 템플릿 ID 정하기 (D, E, F...)
- [ ] 템플릿 이름 정하기 (예: "D 쇼핑형", "E 이벤트형")
- [ ] 카테고리 분류 (표지 / 내지 / 아웃트로)
- [ ] 주요 입력 필드 목록 작성
- [ ] 미리보기 레이아웃 디자인 (피그마 등)

**`docs/COMPONENT_RULES.md` 참고하여 체크**

### ImageUploader 사용?
- [ ] 이미지 업로드가 필요한가?
  - [ ] 연출컷만? 누끼컷만? 둘 다?
  - [ ] 위치 조정 기능 필요?

  - [ ] 배경 컬러 → `bgColor` + `_bgColorAuto`
  - [ ] 기타 커스텀 컬러?
- [ ] → 필요하면 `ColorPicker` 섹션 규칙 따름

### 폼 요소
- [ ] 텍스트 입력 필드 (글자 수 제한 필요?)

## 📋 STEP 3: 데이터 구조 설계

**카드 데이터 스키마 설계**

```javascript
// 예시: D 템플릿 데이터 구조
{
  templateId: 'D',
  data: {
    // 공통 (모든 템플릿)
    _img: null,           // 원본 이미지
    _imgType: '연출컷',   // '연출컷' | '누끼컷'
    _imgTransform: {...}, // 위치 조정

    // 컬러 (필요한 것만)
    btnColor: null,
    _btnColorAuto: true,

    // 템플릿 고유 필드
    mainText: '',
    subText: '',
    buttonText: '',
    // ... 필요한 필드들
  }
}
```

- [ ] 공통 필드 정리 (`_img`, `_imgType` 등)
- [ ] 컬러 필드 + 자동 플래그
- [ ] 템플릿 고유 필드

## 📋 STEP 4: HTML 편집 폼 작성

**`renderEditor` 함수에 템플릿 분기 추가**

### 작성 위치
- 파일: `public/kakao-crm-builder.html`
- 함수: `renderEditor()` 또는 템플릿별 개별 함수

### 작성 구조

```javascript
if (templateId === 'D') {
  return `
    <!-- 제목 -->
    <h2>D 템플릿 편집</h2>

    <!-- 이미지 업로드 -->
    ${imageUploaderHTML(...)}

    <!-- 텍스트 입력 -->
    <label>메인 텍스트</label>
    <input ... maxlength="20">

    <!-- 컬러피커 -->
    ${iroCP({...})}

    <!-- 기타 필드 -->
  `;
}
```

- [ ] HTML 구조 작성
- [ ] 공통 컴포넌트는 규칙대로 재사용
- [ ] 글자 수 카운터 추가 (텍스트 필드)
- [ ] `esc()` 함수로 XSS 방지

## 💡 팁

### 빠른 개발 순서
1. 기존 템플릿(A, B, C) 코드 참고하여 **복사 + 수정**
2. 비슷한 구조라면 A 템플릿 복사가 빠름
3. 공통 컴포넌트는 **최대한 재사용** (iroCP 등)

### 디버깅
- 브라우저 개발자 도구(F12) Console 확인
- 렌더링 안 되면 `cards` 배열과 `selectedIdx` 확인
- 컬러피커 문제는 `_iroInstances` 확인

### 성능
- 이미지 업로드 시 base64 크기 주의 (1-2MB 권장)
- 많은 카드 추가 시 렌더링 속도 확인
